# 体重记录 App 需求文档

## 1. 产品概述

体重记录 App 用于帮助用户持续记录每日早晚体重变化，并通过曲线图展示趋势，辅助用户观察体重管理效果。

## 2. 目标用户

- 正在减脂、增肌或维持体重的人群
- 需要长期监测体重变化的用户
- 希望区分早晚体重差异、观察趋势的人群

## 3. 核心功能

### 3.1 早晚体重记录

用户可以每天分别记录：

- 早晨体重
- 晚上体重

记录字段：

| 字段 | 说明 |
|---|---|
| 日期 | 默认当天，可手动选择 |
| 记录类型 | 早晨 / 晚上 |
| 体重 | 支持 kg，保留 1 位小数 |
| 备注 | 可选，如饮食、运动、睡眠等 |

功能规则：

- 同一天早晨体重只能有一条记录
- 同一天晚上体重只能有一条记录
- 已存在记录时，再次输入应进入编辑模式
- 支持修改和删除历史记录
- 输入体重时需校验合理范围，例如 20kg - 300kg

### 3.2 体重曲线图

系统根据用户记录生成体重变化曲线。

曲线图要求：

- 支持展示早晨体重曲线
- 支持展示晚上体重曲线
- 支持同时展示早晚体重对比曲线
- 横轴为日期，纵轴为体重
- 点击曲线点可查看具体日期、体重和备注
- 长按曲线点可编辑该条记录
- 空状态引导文案为"记录第一天的体重吧"

时间范围筛选：最近 7 天 / 30 天 / 90 天 / 自定义时间范围

### 3.3 首页概览

- 今日早晨体重、今日晚上体重
- 最近一次记录体重及相比上一条的变化值
- 最近 7 天体重趋势简图
- 快捷记录入口

### 3.4 历史记录

- 按时间倒序展示，支持查看早晨/晚上记录
- 支持编辑、删除记录
- 支持按日期筛选

## 4. 页面结构

| 页面 | 主要模块 |
|---|---|
| 首页 | 当前体重卡片、今日记录入口、最近趋势图、最近记录列表 |
| 记录页 | 日期选择、早晚切换、体重输入、备注、保存 |
| 趋势页 | 体重曲线图、时间范围筛选、早晚/对比切换、数据点详情 |
| 历史页 | 历史记录列表、日期筛选、编辑/删除操作 |

## 5. 数据模型

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 记录 ID |
| date | string | 日期，格式 YYYY-MM-DD |
| period | enum | morning / evening |
| weight | number | 体重 |
| note | string | 备注 |
| createdAt | datetime | 创建时间 |
| updatedAt | datetime | 更新时间 |

## 6. 非功能需求

- **易用性**：记录流程简短、快捷记录入口、数字键盘适配移动端
- **数据安全**：本地保存或云端同步、二次确认删除、安全传输
- **性能**：首页加载 < 1 秒、1 年数据量内曲线流畅、分页加载历史记录

## 7. 后续可扩展功能

目标体重设置、BMI 自动计算、周报/月报、体脂率记录、围度记录、数据导出、记录提醒

## 8. 验收标准

- 可成功新增、编辑、删除早晚体重记录
- 同一天同一时段不会产生重复记录
- 曲线图正确展示早晨、晚上及对比趋势
- 支持按 7 天、30 天、90 天和自定义范围查看趋势
- 历史记录与曲线图数据保持一致

## 9. 统计 API（补充）

### 9.1 获取统计数据

`GET /weight-records/stats?startDate=2026-04-01&endDate=2026-05-02`

说明：返回当前用户在日期范围内的统计数据。

请求参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| startDate | string | 否 | 无 | 起始日期，格式 YYYY-MM-DD |
| endDate | string | 否 | 无 | 结束日期，格式 YYYY-MM-DD |

响应字段（WeightStats）：

| 字段 | 类型 | 说明 |
|---|---|---|
| avgMorningWeight | number \| null | 平均早体重 |
| avgEveningWeight | number \| null | 平均晚体重 |
| minWeight | number \| null | 最低体重 |
| maxWeight | number \| null | 最高体重 |
| change | number \| null | 体重变化量（最后有效体重 - 第一有效体重） |

空数据响应：

```json
{
  "avgMorningWeight": null,
  "avgEveningWeight": null,
  "minWeight": null,
  "maxWeight": null,
  "change": null
}
```

计算规则：
- 所有统计结果保留 1 位小数
- 平均值只统计非空值，缺失值不当作 0
- 最低/最高体重从早体重和晚体重的全集中计算
- 同一天同时存在早/晚体重时，第一有效体重优先取早体重，最后有效体重优先取晚体重
